# 📖 Overview

This project simulates a real-world Library Management System using PostgreSQL. It manages books, members, employees, issued books, returned books, and branch information. The project focuses on writing efficient SQL queries, designing relational databases, and solving business problems using SQL.

---

# 🎯 Business Problem

Libraries require an efficient system to manage books, members, issued books, returns, and employee operations. Manual management becomes inefficient as records increase.

This project demonstrates how SQL can automate these operations while providing meaningful business insights.

# 🎯 Objectives

- Design a relational database for a library.
- Maintain book and member records.
- Track issued and returned books.
- Solve business questions using SQL.
- Practice advanced SQL concepts.
- Improve database querying and reporting skills.

# 📊 Business Questions Solved

This project answers business questions such as:

- Which books are currently issued?
- Which books are overdue?
- Which members borrowed the most books?
- Which category is most popular?
- How many books are available?
- Which employee issued the highest number of books?
- Monthly issue trend analysis.
- Branch-wise issued books.
- Return status analysis.
- Book availability report.
- Member activity analysis.
- Most rented books.
